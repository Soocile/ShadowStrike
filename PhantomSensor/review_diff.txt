       audit(behavioral): ThreatScoring + BehaviorEngine principal-level security audit
    
       CRITICAL fixes:
       - ProcessNotify: TsCalculateScoreInPlace now queries correct engine
         (g_ProcessMonitor.ThreatScoringEngine, not driver-wide instance)
       - BehaviorEngine: removed dead g_ScoringEngine (never initialized)
    
       HIGH fixes:
       - BepRemoveChainFromHash/BepRemoveProcessContextFromHash: eliminated
         double-RemoveEntryList that caused LIST_ENTRY corruption; callers
         now explicitly handle main-list removal before hash-table removal
       - BeEngineReleaseChain/ReleaseProcessContext: added main-list removal
         in immediate-cleanup paths after hash-removal refactor
       - BepCleanupStaleProcessContexts: now calls BepRemoveProcessContextFromHash
         before freeing, preventing hash-table dangling pointers / use-after-free
       - TsRunMaintenancePass/TsPurgeStaleContexts wired into periodic maintenance:
         driver-wide engine in BehaviorEngine worker thread,
         ProcessMonitor engine in PnpCleanupTimerCallback
    
       MEDIUM fixes:
       - BepIsLolBin/BepIsScriptHost: case-insensitive suffix match via _wcsnicmp
         (was case-sensitive wcsstr, would miss CMD.EXE/PowerShell.exe)
       - Hoisted wcslen(ImagePath) outside loops (computed once, not per-iteration)
       - Removed duplicate msiexec.exe from LOLBin list
       - Removed unconditional TotalRuleMatches increment (was inflating stats)
